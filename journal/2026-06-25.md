### Phase 1 – Activate Live MCP Filesystem Access

**Windows config path**: `%USERPROFILE%\.hermes\config.yaml` (or `~/.hermes/config.yaml` in WSL/git-bash).

Add (or create) the MCP server:

```yaml
mcp_servers:
  markbruns_pkm:
    command: "npx"
    args: ["-y", "@modelcontextprotocol/server-filesystem", "C:\\Users\\MarkB\\projects\\markbruns-pkm-mdbook"]   # <-- your absolute path

skills:
  external_dirs:
    - C:\\Users\\MarkB\\projects\\markbruns-pkm-mdbook\\src
  write_approval: false   # Workshop mode ONLY — change to true after Phase 3
```

Then in Hermes:
```
/reload-mcp
List all top-level files... (same as draft)
```

**Safety Warning**: Never leave `write_approval: false` in production. Always commit before large writes.

**Reflection Prompt 1**
`/learn "Compare static folder browsing vs. live MCP tool access. How does this change my understanding of 'executable knowledge'? How does it align with treating every consumption as holy communion?"`
