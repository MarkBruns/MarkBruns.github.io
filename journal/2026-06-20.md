### 3. The Daily Rhythm (Non-Negotiable)

**04:00 – 05:00** Holy Hour
Prayer, Scripture reading (full Christian Bible), Trinitarian examen, intercession, and listening. No devices except what is required for the Word.

**05:00 – 05:30** First communion (breakfast) + explicit prayer of transformation.

Then 15 blocks of 30 minutes each. Each block follows the same micro-rhythm:
- 25 minutes of focused technical work
- 4 minutes of Tabata-style movement or martial drills
- 1 minute of reset + short prayer

**Sample Block Allocation (adjust to your portfolio)**

| Blocks | Theme                              | Purpose |
|--------|------------------------------------|-------|
| 1–3    | Deep technical investigation       | Root-cause work on highest-leverage problem |
| 4–6    | Portfolio stewardship & writing    | Update 101 Portfolio, journal, public artifacts |
| 7–8    | Agentic prospecting                | Scanning for better opportunities |
| 9–10   | PKM / Second Brain maintenance     | Capture, organize, link |
| 11–12  | Skill or tool improvement          | Making the agent and systems better |
| 13–14  | Review + pruning                   | Close loops, sunset weak ventures |
| 15     | Weekly rhythm & planning           | Set up the next days |

**Meals** (lunch, dinner) are always treated as communion with explicit prayer.

Evening ends with a short examen and wind-down. No new loops after a reasonable hour.

Hermes Agent is not a chatbot. It is your **autonomous operator and thought partner**.

You give it the Rule of Bruno as its soul, and it becomes the entity that protects your attention, surfaces opportunities, prunes low-value work, and turns intent into executed reality.

### 5. Installing Hermes Agent (Desktop)

1. Open a terminal and run the official installer:
   ```bash
   curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
   ```

2. Run the setup wizard:
   ```bash
   hermes setup
   ```

3. Choose your model/provider (Grok, Claude, OpenRouter, local models, etc.). The system is provider-agnostic.

4. Create your `SOUL.md` file in the Hermes root directory:
   - Location: `C:\Users\YourName\AppData\Local\hermes\SOUL.md` (Windows) or `~/.hermes/SOUL.md` (macOS/Linux)
   - Paste the full BrunoSoul persona (or your adapted version).
   - Hermes will load this file on every new session.

5. Recommended first commands:
   ```bash
   hermes profile show default
   hermes tools list
   hermes skills list
   ```

6. Start working:
   ```bash
   hermes
   ```
   Or use `/new` inside a session to begin fresh under the Rule.

### 6. The 101 Portfolio Lifestyle

Your career can never be a single job -- that's IDIOT-THINK. Your workflow has to be a diversified portfolio of 101 micro-ventures and assets.

The staging document lives at a public mdBook (example: `101PortfolioLifestyle.html`). It contains:
- Active high-leverage ventures
- Needs-work items
- Back-burner projects
- Sunset candidates
- Operational debt list

Every Monday (or equivalent) you run a portfolio review block:
- Promote anything that is producing fruit
- Ruthlessly prune or archive anything that is not
- Update the public-facing version so the portfolio is always current

This becomes your professional brand: a living, public demonstration of disciplined, high-quality technical work.

### 7. Agentic Prospecting & Brand Building

A core daily discipline is **agentic opportunity prospecting**.

Hermes (or you, using Hermes) runs regular scans for:
- Interesting technical problems on GitHub, arXiv, forums, Discord, X, company blogs
- Open-source issues that match your root-cause investigation brand
- Research questions that need rigorous investigation
- Potential clients or collaborators who value depth over meetings

You never apply for jobs in the traditional sense. Instead, you **ship visible artifacts** (journal entries, technical write-ups, solved problems, open-source contributions) that demonstrate your capability. The brand becomes: “This person finds root causes that others miss and explains them clearly.”

All prospecting happens in the virtual realm. No conferences. No travel. No unnecessary meetings.

### 8. Second Brain & Personal Knowledge Mgmt (PKM) Integration

Use a PARA-based mdBook system (Projects, Areas, Resources, Archives) with heavy emphasis on the 15–20 categories that actually matter to a technical problem-solver (Christian Spiritual Health, Time Optimization, AI, arXiv categories, etc.).

### 9. Execution Standards

- **Vague is rejected.** Every task must have clear scope and next action.
- **Evidence required.** When Hermes pushes back, it must show data, Scripture, or clear risk.
- **Ship weekly.** At least one visible artifact per week that advances the portfolio or brand.
- **Close loops.** Never open more ventures than you are actively stewarding.
- **Protect the 45 hours.** Anything that threatens the rhythm is questioned aggressively.
