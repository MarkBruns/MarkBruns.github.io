# Production-Grade AI App Architectures 

These examples of production-grade AI app architectures focus on agentic RAG, multi-agent systems, or LLM-powered apps built for scalability, maintainability, and real-world deployment (often using LangGraph, FastAPI, Docker, evals/monitoring like Langfuse/Prometheus, and modular folder structures). I prioritized ones with explicit layered/modular designs mirroring the 9-layer breakdown. 

It is worthwhile to [explore the RECENTLY updated GitHub repositories](https://github.com/topics/agentic-ai?o=desc&s=updated), especially focusing one's attention on those with many stars or forks -- these are those that are selected on the basis of their forks/stars. 

1. **https://github.com/FareedKhan-dev/production-grade-agentic-system**  
   Core 7-layer production agentic system with `app/api/`, `core/langgraph/` (agents), `prompts/`, `services/`, `evals/`, Grafana/Prometheus observability, security (rate limiting/sanitization), and metrics—explicitly modeled after production AI patterns.

2. **https://github.com/jamwithai/production-agentic-rag-course**  
   Full production RAG course project (`arxiv-paper-curator`) with `src/routers/`, `services/` (agents/cache/LLM), models/schemas, Docker, monitoring, and hybrid search—built as a complete scalable pipeline.

3. **https://github.com/GiovanniPasq/agentic-rag-for-dummies**  
   Modular LangGraph agentic RAG with query clarification, multi-agent map-reduce retrieval, self-correction, parent/child chunking, and detailed project structure (config, utils, chunker, workflow).

4. **https://github.com/hoangsonww/Agentic-AI-Pipeline**  
   Production-ready agentic pipelines (RAG + coding agents) with separate folders for Agentic-RAG-Pipeline, tools, schemas, MCP server, and FastAPI—includes observability and self-critique.

5. **https://github.com/zenml-io/template-rag**
