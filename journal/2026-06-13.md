# Exploration of **Agentic OS** and GRIT / Jujutsu

*An infrastructure or architectural layer designed specifically for AI agents, enabling them to operate autonomously, coordinate, and scale reliably—much like a traditional OS manages apps and resources, but tailored for OBSERVABILITY of goal-driven, reasoning entities.*



An agentic OS typically handles:
- Resource management (memory/context, scheduling, tools)
- Orchestration of multiple agents
- Persistent state and learning
- Guardrails/security
- Observability and human oversight

Examples include conceptual frameworks (e.g., from Slack or MindStudio discussions), dedicated projects like **AIOS** (with its kernel abstracting LLM/memory/storage/tools/scheduler), VAST AI OS, or Hermes as a practical "always-on" agent layer in broader agentic systems.

The goal is turning collections of LLMs/tools into coordinated, reliable "workforces" that plan, act, remember, and collaborate without constant human intervention or fragility in complex workflows.

**Subagent orchestration** is a core pattern within agentic systems: a primary (or "conductor") agent delegates subtasks to specialized, often isolated sub-agents that run in parallel or sequence. This reduces context bloat, enables specialization, supports parallelism, and improves efficiency/reliability for multi-step or branched tasks.

In practice:
- Sub-agents get their own conversation history, tools/terminals, or execution environments.
- Orchestration involves task decomposition, delegation, result aggregation, and coordination (hierarchical supervisor or more decentralized).
- Background/independent processes (e.g., long-running cron-like tasks) run alongside without blocking the main flow.

This mirrors OS process/thread management but at the agent level.

**Minimalist kernels** in this context are lightweight abstraction layers or full kernels optimized for AI/agent workloads, rather than general-purpose OS kernels. They provide primitives for agent identity, capabilities, scheduling, memory/context management, tool access, and security—often with minimal overhead.

Notable examples:
- **AIOS Kernel** (from agiresearch/AIOS): Abstracts OS resources for agents (LLM, memory, storage, tools, scheduler) with syscall-like dispatching; includes experimental Rust scaffold for traits/implementations in context/memory/storage/tool/scheduler/LLM.
- **OctantOS** (OctantCore): A from-scratch Rust kernel + hypervisor built specifically for autonomous AI agents, with kernel-level primitives for identity, capability boundaries, TTL enforcement, and auditing. It emphasizes a 10-layer security stack because "application-layer trust is insufficient for autonomous agents."
- Others like OneClaw (lightweight Rust agent kernel for edge/IoT), AutoAgents (Rust runtime for safe/production agents), or OpenFANG (Rust-based "Agent OS" with kernel crate for orchestration/RBAC/scheduling).

These prioritize **memory safety, security by design, performance, and concurrency**—ideal for agents that act with real autonomy.

### Rust-Based Systems: Memory Safety, Security, Speed, and Observability
Rust shines here due to its ownership model (memory safety without GC overhead), fearless concurrency (async, no GIL like Python), and suitability for systems programming. Developers are moving agentic code to Rust (or hybrid Python+Rust via PyO3) for production reliability, especially in concurrent/multi-agent scenarios.

Key advantages for agentic systems:
- **Memory-safe & secure**: Prevents common bugs/exploits; supports capability-based security and kernel-level enforcement (e.g., OctantOS makes permissions/audit first-class kernel primitives).
- **Fast & efficient**: Low-overhead runtimes, excellent for parallel sub-agents/background tasks, edge deployment, or high-throughput orchestration.
- **Inherently designed observability/minimalist tracing/logging/monitoring**: Many Rust agent frameworks/kernels build in structured logging, tracing, audit trails, metrics, and real-time status as core features (not bolted-on). This supports **test-in-production** mindsets—run live, monitor/audit/debug/iterate safely without halting systems. Examples include kernel audit primitives, budget/telemetry enforcement, progress tracking, and composable tracing.

Projects like AutoAgents, Symbiont (zero-trust Rust framework), mini-agent frameworks, and ActonAI (actor model with kernel supervisor) exemplify this.

Broader trends include Rust microkernels/experiments for AI-native OSes (optimizing for large models, GPU/CPU sync, model lifecycle) and discussions around rewriting performance-critical agent parts in Rust.

### Hermes Dev Community: Key Recent Developments (Early-Mid June 2026)
**Hermes Agent** (Nous Research, open-source MIT, GitHub: NousResearch/hermes-agent) is a prominent practical implementation: a self-improving, persistent AI agent that "grows with you." It features built-in learning loops (creates/refines skills from experience), long-term memory/user modeling, multi-platform gateways (Telegram, Discord, Slack, etc.), flexible model providers, sandboxed execution, and continuous operation (VPS, serverless, desktop).

It aligns closely with agentic OS concepts as an always-on autonomous layer with orchestration capabilities. Core is primarily Python (with sandboxes like Docker for isolation/security), plus TypeScript/JavaScript for the desktop app. No core Rust in Hermes itself, but it operates in an ecosystem increasingly exploring Rust for underlying runtimes/kernels, and community members discuss deeper system integrations (e.g., agent access to Linux kernels or building custom interfaces).

**Recent key developments** (focusing on ~June 1–12, 2026 activity, especially desktop and observability):
- **Desktop App Major UX and Observability Upgrades** (very active on June 12): Native app for macOS/Windows/Linux with live subagent windows, composer status stack, editable UI elements, and better real-time monitoring. Refactors include ephemeral port discovery. This directly enhances visibility into complex agent workflows.
- **First-Class Subagent + Background Tasks Observability Demo**: A highlighted community demo (quoted by lead engineer @Teknium) shows the desktop UI breaking down tasks (e.g., "scan repo" → subtasks like count .py files, check git status, summarize), spawning parallel sub-agents and independent background processes, with real-time progress, thinking steps, results, and independent execution (e.g., background sleep/echo loops running while sub-agents work). Sub-agents and background tasks run independently with echoed progress and final outputs visible. This is a concrete step toward richer Agentic OS UX.
- **Integrations and Polish**: Production-grade WhatsApp Business Cloud integration (secure webhooks, media/voice, read receipts, typing indicators, interactive approvals). Other gateway fixes (Discord, Telegram). New `/credits` command and billing integration. Profile Builder updates for easier setup (identity, models, skills, servers). Docker/Nix build improvements and regression guards.
- **Self-Improvement and Production Features**: Trajectory compression/logging for learning loops and training data pipelines. Commands for usage/insights/compression. Cron-based scheduling for unattended automations (reports, backups). Security features like command approval prompts (with `--yolo` bypass option) and container isolation.
- **Community Momentum**: Users sharing real workflows (e.g., complex government/project management at SBA using Hermes for iteration/documentation; personal "AI OS" backbones with voice briefings, R&D, coding, monitoring, multi-model balancing, and persistent memory/skills/cron). Discussions on deeper integrations like giving agents Linux kernel access or building dynamic "liquid interfaces." Calls for more user story spotlights. Community extensions (e.g., computer-use tools, bridges like HermesClaw).

Hermes emphasizes **practical test-in-production**: Run continuously on servers, monitor via desktop/UI/logs/insights, use trajectories for self-improvement, schedule background tasks, and apply granular approvals/budgets. Recent observability enhancements (status stacks, live subagent/bg task views, structured logging like `hermes_logging.py`) directly support this by making internal agent behavior inspectable in real time.

**Tying It Together**: Hermes provides accessible, feature-rich agent orchestration and growing observability (especially subagents/background via desktop), fitting into the broader agentic OS vision. Complementary Rust-based minimalist kernels/runtimes (OctantOS, AIOS experimental parts, AutoAgents, etc.) offer stronger foundations for memory safety, inherent security/auditability, performance, and built-in monitoring—ideal for production-grade or edge deployments where test-in-prod requires robust, observable primitives without Python's limitations.

The space is rapidly evolving toward composable systems: high-level agent frameworks like Hermes on top of (or integrated with) safer/faster Rust kernels and observability layers. This enables more reliable autonomous operation with visibility for iteration and trust.

For hands-on exploration:
- Hermes: Install via official script; try the desktop app for subagent/bg observability demos.
- AIOS or OctantOS repos for kernel-level experiments.
- Watch for more community integrations bridging these (Rust runtimes powering or extending Hermes-like agents).

This represents a shift from brittle single-shot agents to robust, observable, self-improving agentic systems.
